## üè¢ Amazon

### Interview Style
- Focus on **practical problem-solving**
- Values **clear communication** of approach
- Expects **working code** (less emphasis on proofs)
- Often asks **classic interval problems**

### Most Common Problems

| Problem | Frequency | Notes |
|---------|-----------|-------|
| Merge Intervals | ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê | Must know perfectly |
| Meeting Rooms II | ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê | Very common |
| Task Scheduler | ‚≠ê‚≠ê‚≠ê‚≠ê | Common in SDE2+ |
| Jump Game | ‚≠ê‚≠ê‚≠ê‚≠ê | Classic greedy |
| Gas Station | ‚≠ê‚≠ê‚≠ê | Tests circular thinking |

### Amazon Tips

1. **Explain your approach first** - Don't jump into code
2. **Think about edge cases** - Empty inputs, single elements
3. **Optimize for readability** - Clean code matters
4. **Relate to real-world** - "Like scheduling delivery routes"

### Sample Dialogue

```
Interviewer: "Design a meeting room scheduler"

You: "So we need to find the minimum number of rooms needed 
for all meetings. I'll sort meetings by start time and use 
a min-heap to track when rooms free up. This gives O(n log n) 
time and O(n) space for the heap."
```

## üîµ Google

### Interview Style
- May ask for **proof of correctness**
- Expects **optimal solution**
- Often combines patterns
- Tests **edge case thinking** deeply

### Most Common Problems

| Problem | Frequency | Notes |
|---------|-----------|-------|
| Jump Game II | ‚≠ê‚≠ê‚≠ê‚≠ê | Optimal solution expected |
| Candy | ‚≠ê‚≠ê‚≠ê‚≠ê | Two-pass reasoning |
| Course Schedule III | ‚≠ê‚≠ê‚≠ê | Heap + greedy |
| Minimum Arrows | ‚≠ê‚≠ê‚≠ê | Interval counting |
| Partition Labels | ‚≠ê‚≠ê‚≠ê | Last occurrence |

### Google Tips

1. **Be ready to prove correctness** - Exchange argument
2. **Optimize before coding** - O(n) if possible
3. **Discuss trade-offs** - Time vs space
4. **Handle follow-ups** - "What if input is streaming?"

### Sample Dialogue

```
Interviewer: "Why is your greedy approach optimal?"

You: "At each step, we make the locally optimal choice of 
[X]. This is globally optimal because any alternative choice 
can be swapped with ours without improving the result - 
an exchange argument. Specifically, if we chose [Y] instead, 
we could always swap to [X] since [reason]."
```

## üî∑ Meta (Facebook)

### Interview Style
- Emphasizes **bug-free coding**
- Tests **code execution** (fewer syntax errors)
- Values **quick problem-solving**
- Often has **45-minute slots** (less time)

### Most Common Problems

| Problem | Frequency | Notes |
|---------|-----------|-------|
| Merge Intervals | ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê | Standard problem |
| Insert Interval | ‚≠ê‚≠ê‚≠ê‚≠ê | Variation |
| Non-overlapping | ‚≠ê‚≠ê‚≠ê‚≠ê | Sort by end |
| Task Scheduler | ‚≠ê‚≠ê‚≠ê | Common |
| Two City Scheduling | ‚≠ê‚≠ê‚≠ê | Optimization |

### Meta Tips

1. **Write clean, bug-free code** - They run your code
2. **Think about types** - Python: list vs set vs dict
3. **Test before submitting** - Trace through example
4. **Be concise** - Less time for discussion

### Sample Dialogue

```
Interviewer: "Merge overlapping intervals"

You: "I'll sort by start time, then iterate and merge if 
current start <= previous end. Let me code this..."

[Write code, then:]

"Let me verify with example [[1,3],[2,6],[8,10]]...
After sorting: same. Merge (1,3) and (2,6) ‚Üí (1,6).
(8,10) doesn't overlap. Result: [[1,6],[8,10]]."
```

## üéØ Preparation by Company

### Preparing for Amazon
1. Master interval merging perfectly
2. Practice Meeting Rooms II with heap
3. Review Jump Game and Gas Station
4. Focus on communication and edge cases

### Preparing for Google
1. Understand proofs (exchange argument)
2. Practice optimal solutions (not just working)
3. Review two-pass greedy (Candy)
4. Prepare for follow-up questions

### Preparing for Meta
1. Write bug-free code (practice on paper)
2. Master interval problems quickly
3. Time yourself (45 min slots)
4. Test your code before "running"

### Preparing for Microsoft
1. Practice explaining thought process
2. Start simple, then optimize
3. Cover edge cases thoroughly
4. Show code quality (naming, structure)
